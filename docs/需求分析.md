# 需求分析

> 学科知识整合智能体 — 子问题分解与分析

## 1. 知识点粒度定义

- 知识点定义为教材章节中的核心概念、定理、方法或现象
- 每个知识点包含：名称、定义（1-3 句）、分类、所属章节、页码
- LLM 提取时限制每章节 3-8 个最核心知识点，避免过度提取

## 2. 重复判定标准

- **完全匹配**: 知识点名称完全相同（最高置信度 0.95）
- **语义相似**: Embedding 余弦相似度 ≥ 0.82 判定为重复
- **双重确认**: Embedding 初筛 + LLM 复核（P1 进阶）

## 3. 教学连贯性保障

- 关系边保留：合并节点时保留所有入边/出边关系
- 前置依赖链：prerequisite 关系不会被删除
- 压缩策略：优选定义最完整、出处最权威的版本

## 4. 压缩比计算方式

```
压缩比 = 整合后保留内容字数 / 原始教材总字数
目标：≤ 30%
```

- 字数计算基于章节正文内容
- 严格控制在 30% 以内

## 5. RAG 分块策略选择依据

- **分块大小**: 500-800 字（平衡语义完整性与检索精度）
- **重叠**: 50-100 字（sliding window，防止截断）
- **选择理由**: 中文教材以段落为自然语义单元，500-800 字约 2-3 个段落
